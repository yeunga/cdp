Class Detail
voview(input, xsltdir, objectName, widgetIDprefix)
Creates a new voview object for displaying a VOTABLE.
Parameters:
{Object} input
Parameters to be passed to the votable object. One of the properties: tree, string, url, form must be specified.
{XML DOM Object} input.tree Optional
XML DOM Object of a VOTABLE.
{string} input.string Optional
String containing VOTABLE.
{string} input.url Optional
URL pointing to a VOTABLE.
{string} input.form Optional
Name of a form used for specifying the VOTABLE.
{HTML DOM Element} input.searchparam Optional
Name of input element in the form containing the VOTABLE. Required if input.form is specified.
{string} xsltdir
Directory on the server to look for XSLT files.
{string} objectName
The name of the global variable pointing to this instance of the voview class. This will be used for referencing the proper JavaScript calls when rendering the HTML table.
{string} widgetIDprefix
Prefix for the HTML attribute IDs where the various sub-widgets of the HTML table display will be placed. For example, when displaying the main table containing the data, the HTML element with an ID of {prefix}.table would be searched for, and the table placed at this location in the HTML document. See the documentation on voview.renderer.render for the list of sub-widgets.

{integer[]} findRows(selectCriteria)
Find the indexes of the rows in the original VOTABLE which match the selection criteria.
Parameters:
{string|function} selectCriteria
If a string, use it to match against the contents of the VOTABLE row. The contents of the VOTABLE XML row will be searched and the row will be selected if it contains the input string. If a function, then a function which will be called for each row in the XML VOTABLE, with an XML DOM object of the row as its only argument. The function should return a boolean indicating whether the row should be selected or not.
Returns:
{integer[]} An array of indexes which correspond to the positions of the rows in the original VOTABLE.
